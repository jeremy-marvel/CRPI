ld_comm_client Constructor for ld_comm object.Creates log file for transmitted poses, initializes the
address of the ld ARCl server, and password for logging into the server
The values passed into this function are determined by the Adept ld Configuration Parameters.
These can be changed from the vehicle under Interface Options in Mobile Planner.
Then, when calling this constructor, the new parameters can be entered.

Parameters
[in]arcl_addrIP address of the ld ARCL server
[in]arcl_portport number used when connecting to ARCL server
[in]arcl_passwdpassword given to the ARCL server to login.
[in]logboolean used to enable logging of ARCL data.

int ld_comm_client::arcl_dsfv_pose ( ld_msg_pose *  pose)

Sends multiple commands to get the current X, Y, and Theta pose of the vehicle,
plus another to get the timestamp. Parses the response and then groups
the data to store within a datastructure. Access pose and status
data within the ARCl DataStore. This not the best way to monitor the ld
as multiple commands issued in sequence are not guaranteed to return an atomic
response.

Parameters
[out]posedatastructure to hold the pose and timestamp that is returned.
Returns
An integer indicating the error status of the function. A negative return value indicates a ulapi socket error.

◆ ld_connect()

int ld_comm_client::ld_connect ( )

Connects to the ARCL server, sends the password login, and discards extra text sent back from the server.
This function establishes the socket connection to the ld ARCL server.
This includes opening the client socket, sending the password to login to the ld,
and parsing extra input generated by the ARCL server.
