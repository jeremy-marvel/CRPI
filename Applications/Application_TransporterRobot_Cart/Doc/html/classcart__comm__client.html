◆ remote_command()

int cart_comm_client::remote_command ( char *  cmd)

Performs an action depending on the message passed to the function. Intended to process messages sent from the vehicle server process. These actions include updating the current operating status of the cart and forwarding information back to the vehicle server process among other things.

Parameters
[in]msgA character buffer where the leading integer is a single digit. Acceptable commands include:
"rcstat" – Forwards current status in the format "%d%s" to the vehicle server process. The integer is the current operating status while the string is the next dock point destination. "rcstatinit" – Forwards initial status info of the format "%d%s:%s" where the first integer is the initial operating status, the first string is the dock point that the cart starts at, and the second string is the first dock point destination. "rcupdate0"– Updates status of cart to 0 (Ready)
"rcupdate1"– Updates status of cart to 1 (Waiting)
"rcupdate2"– Updates status of cart to 2 (Busy)
Returns
An integer indicating the error status of the function. Returns a negative number if an error occurred.

bool  level_pause

Control code for performance test method. Sends manipulator to an initial stage pose for safe access to the RMMA, then performs pose calculations based on the current vehicle position to adjust for docking offset.
Finally, calls function to perform edge registration (does not use continuous move, only moves incremented by steps) and spiral search verification. This version was one of several edge registration methods tested with the OTS and was measured on 06-28-2019.

Parameters
[in]ur_robotA pointer to a CRPI datastructure that manages the connection to the UR5 robot URScript server. Needed to send move commands to control the arm.
[in]level_pauseFlag to enable pausing after extending feet to allow for level measurements.

void cart_comm_client::test_square_auto_update ( CrpiRobot< CrpiUniversal > *  ur_robot,

int *  bisect_stat

Control code for performance test method. Sends manipulator to an initial stage pose for safe access to the RMMA, then performs pose calculations based on the current vehicle position to adjust for docking offset.
Finally, calls function to perform bisect registration and spiral search verification. Additionally, records the measured actual position of the bisect reflectors, which can be used to update the initial search positions stored in the configuration file.

Parameters
[in]ur_robotA pointer to a CRPI datastructure that manages the connection to the UR5 robot URScript server. Needed to send move commands to control the arm.
[out]update1A pointer to a PM_CARTESIAN datastructure that stores the measured actual position of the first bisect reflector after searching. Is not updated if bisect search fails.
[out]update2A pointer to a PM_CARTESIAN datastructure that stores the measured actual position of the second bisect reflector after searching. Is not updated if bisect search fails.
[out]bisect_statA pointer to a integer used for error checking. Set to 1 if bisect is successful or 0 if bisect is unsuccessful.
